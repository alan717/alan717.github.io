编译llvm

目标

编译LLVM

思路

  1. 从官网下载源码包
  2. 配置编译脚本，根据构建脚本提示安装编译依赖包
  3. 编译ing
  4. 部署编译好的库文件

详细步骤

1.首先从官网下载到llvm-7.0.1的源码包，解压确定构建系统

从图中可以确定这是CMake构建系统

2.CMake构建 为了不污染源码文件夹

mkdir build

cd build

cmake ..

这样

然后下来就是 make 命令

根据机器的性能等待编译结束， make install 默认安装位置为/usr/local/lib/目录

编译成功后就是需要的静态库文件，之后就需要链接这些文件包含在工程中，头文件默认是在/usr/local/include/

  1. 成功后就能用#include来引入llvm头文件了。

总结

llvm查看构建系统，不污染代码的情况下进行构建;

可能自己的机器不满足依赖需要，通常cmake会报告依赖错误信息，直接安装错误信息报告出来的包名就可以了。

总体难度不大。

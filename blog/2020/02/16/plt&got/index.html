PLT&GOT表ELF

术语

Global Offset Table ->GOT Procedure Linkage Table->PLT GOT表一般存储的是全局的符号和变量

基本认知

对应.got.plt表，没绝对地址则跳plt对应函数的过程链接函数这个函数会首先跳保存在got中的表项地址，如果got没有，会从got跳到got.plt下边的汇编函数，也就是过程链接函数然后设置需要的参数跳到*dl_time_resolve()，然后dl_time_resolve()函数会找到对应函数的地址并保存在got中，下一次调用这个函数jmp got@func时候就不会跳回到got.plt表下边的指令了。

详细实现

ELF将GOT拆成两个部分".got",".got.plt"，其中.got保存全局变量引用的地址，.got.plt中保存函数引用地址。所有对于外部函数的引用都分离出来放在.got.plt中。它的前三项有特殊用途：

  • 第一项保存的是.dynamic的地址，这个段描述了本模块动态链接相关信息
  • 第二项保存的是本模块ID
  • 第三项是_dl_runtime_resolve()的地址。

其中第二项和第三项由动态链接器在装载共享模块时负责将他们初始化。.got.plt的其余项分别对应每个外部函数的引用。 

.got.plt:
Address of .dynamic
Module ID "Lib.so"
_dl_runtime_resolve()
import func1
import func2
...
...

PLT结构也有点不同，为了避免代码的重复，ELF把最后两条指令放在了PLT中的第一项。 

PLT0:
push *(GOT + 4)
jump *(GOT + 8)
...
bar@plt:
jmp *(bar@GOT)
push n
jump PLT0

PLT在ELF文件中放在.plt段，因为本身是地址无关代码因此，和代码段一起放在可读可执行的Segment被装载进内存。

追加

一般来讲plt hook修改的就是plt第一次跳got表对应函数的地址，修改这个地址即可hook跳走。 

void hook() {
    uintptr_t base_addr;
    uintptr_t addr;
    //1. get the base addr of libpasswd.so
    base_addr = get_base_addr("libpasswd");
    if (0 == base_addr) return;
    addr = base_addr + 0x201020; 
    //2. add the write permisson
    mprotect((void *)PAGE_START(addr), PAGE_SIZE, PROT_READ|PROT_WRITE);
    //3. replace our hook func my_strcmp
    *(void **)addr = my_strcmp; 
    //4. clear the cache
    __builtin___clear_cache((void *)PAGE_START(addr),
            (void *)PAGE_END(addr));
}

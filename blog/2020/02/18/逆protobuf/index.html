逆protobuf

goal

逆一个protobuf二进制bin程序

困难

  • 静态链接protobuf符号被抹去，默认release都是抹除掉，么的说
  • 嵌入式程序的protobuf
  • 需要知道他拿到包数据后的处理流程后运算计算时间签名算法流程，然后根据算法写出对应protobuf数据结构的报文
  • 没有ida可以用，全部linux工具链
  • 未知的荒野，没来过

思路

  1. 首先确定出来prottobuf的版本，sub_cb538函数这个函数是2004001这个版本号，我编译的是3010001.所以我编译的版本高，降级;
  2. 其次，编译出对应的protobuf进行静态链接，进行静态分析
  3. 对应2004001是2.4.1,3010001是3.10.1 我用的3.10.1现在降级到2.4.1
  4. 然后在entry找到对应入口，进行协议解析分析。

实践

编译protobuf

android studio linux 复制到native然后cmake添加即可 具体的cmake不写了太长

生成对应的message文件

用对应2.4.1 版本的protoc生成对应要针对的proto c,cpp文件

分析策略

针对解流饭序列化，匹配找到保存的局部变量，然后根据过程来遍历找到签名算法。
